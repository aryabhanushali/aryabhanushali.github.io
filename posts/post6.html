Structured AI Output with Pydantic & Tool Calling

May 27, 2025 • Arya Bhanushali • AI Tooling, Pydantic, LLMs

My first week at Elevance Health has been packed with learning. I dove deep into how modern AI tools deliver structured outputs, not just text, but JSON objects with defined schemas. This post walks through the three core technologies I explored: Pydantic, tool calling, and structured output from OpenAI's models.

Pydantic: The Backbone of Data Validation

I started by learning Pydantic, a Python library for defining and validating data models. It felt like a mix of type hints and real-world safety nets. With Pydantic, I could create classes that automatically enforce structure and types in my data: 

from pydantic import BaseModel

class Invoice(BaseModel):
    vendor: str
    amount: float
    due_date: str

Instead of writing endless if-statements to check if a field exists or is the right type, Pydantic just raises an error if anything is wrong. Clean, simple, and powerful.

Tool Calling with OpenAI

Next, I learned how to use OpenAI's tool_calling interface. The idea is to let the language model trigger specific functions based on user input, imagine a chatbot automatically calling your get_invoice_summary() or check_contract_terms() function when you ask it a relevant question.

Here's a simplified view of what tool calling looks like: 

{
  "tool_call": {
    "name": "check_invoice",
    "arguments": {
      "vendor": "ACME Corp",
      "amount": 3200.50
    }
  }
}

What amazed me is how natural the interaction feels — the model "knows" when to call tools based on the conversation, and it passes arguments in structured JSON.

Structured JSON Output from LLMs

Finally, I experimented with generating structured responses using OpenAI’s response_format="json" setting. This allowed me to build AI workflows that are not just "chatty," but directly usable by code. For example, getting a response like this: 

{
  "invoice_flagged": true,
  "reason": "Amount exceeds contract limit by 15%"
}

That kind of output is gold when building systems that connect AI with business logic, no extra parsing or guesswork needed.

Final thoughts

This week shifted my perspective on LLMs from "text generators" to "structured reasoning engines." Pydantic gave me tools to validate real-world data, tool calling made AI interactive and responsive, and structured JSON output turned LLMs into collaborators that can slot cleanly into any pipeline.

I'm excited to keep building, and automating, more workflows in the weeks to come.
