How JPEG Compresses Images (Without You Noticing)

May 27, 2025 • Arya Bhanushali • Signal Processing, Compression

After learning about the Discrete Fourier Transform (DFT), I became curious about how these frequency-based ideas show up in real-world tech. And it turns out, one of the most common applications is something we see every day: JPEG image compression.

If you've ever saved a photo as a `.jpg`, you're using a compression method that’s deeply connected to frequency analysis. And just like the DFT, JPEG takes advantage of how signals — in this case, images — can be broken down and represented in smarter, more compact ways.

How Computers Represent Images

At a low level, a digital image is just a grid of pixels. Each pixel stores three numbers: red, green, and blue intensity values (RGB). Each of these is typically 8 bits, or 1 byte — so every pixel takes up 3 bytes of memory.

That adds up fast. A 1000×1000 image already has 3 million bytes (~3MB). But we rarely need all of that raw data — especially because the human visual system doesn’t treat all information equally.

JPEG’s Key Insight: Not All Visual Info Is Equal

JPEG uses a form of lossy compression — meaning it deliberately throws away information. But it does so strategically, by asking: What can humans actually see?

For example, our eyes are much more sensitive to brightness than to subtle color changes. So JPEG converts the image from RGB to a different color space called YCbCr, which separates luminance (Y) from chrominance (Cb, Cr). Then it keeps Y in high detail and compresses Cb/Cr more aggressively — a technique called chroma subsampling.

From Pixels to Frequencies: The DCT

This is where the connection to my last blog post gets exciting. Just like the DFT, JPEG uses a frequency-based transform — the Discrete Cosine Transform (DCT). Images are treated as signals: rows of pixel values that can be analyzed for slow (low frequency) and fast (high frequency) changes.

Here's how JPEG applies it:

  • The image is divided into 8×8 blocks of pixels.
  • Each block undergoes a DCT, converting it from pixel values to 64 frequency coefficients.
  • The top-left value (DC term) is the average brightness, and the rest (AC terms) represent fine-grained detail.

Energy Compaction: Keep the Essentials

DCT has a powerful property called energy compaction: most of the signal's "energy" ends up concentrated in just a few low-frequency coefficients. That means:

  • The most visually important information is in the top-left corner of the DCT matrix.
  • The bottom-right (high-frequency) values are usually small or near zero.
So JPEG compresses by removing many of these high-frequency values — which we often don’t even notice are missing.

Quantization: Rounding with Purpose

After computing the DCT, JPEG applies quantization. Each DCT coefficient is divided by a value from a quantization matrix and then rounded. This reduces precision, and it’s the main source of compression — and quality loss.

But since we care less about high-frequency noise, JPEG rounds or drops those values the most. This allows it to throw away large amounts of data while keeping the image visually intact.

Lossless Compression: Making It Even Smaller

Once quantized, JPEG applies lossless techniques like:

  • Run-Length Encoding (RLE): replaces repeated zeros with a compact representation.
  • Huffman Encoding: uses short binary codes for frequent values, longer ones for rare values.
These reduce file size without losing any more visual quality.

Reconstruction: Back to the Image

When you open a JPEG file:

  1. It’s decompressed with Huffman and RLE decoding.
  2. Quantized values are re-scaled using the same quantization matrix.
  3. An inverse DCT is applied to each 8×8 block to get back pixel values.
  4. The blocks are stitched together to reconstruct the full image.
Some fine detail may be gone, but for most viewing purposes, the image looks almost identical to the original.

Final Thoughts

What I find most fascinating is that JPEG’s magic lies in both math and biology. It doesn’t just compress for the sake of efficiency, it does it in a way that aligns with how we see. By using transforms like the DCT and ideas like energy compaction, it keeps what matters and ditches what doesn’t.
